1. General Configuration

The top-level parameters configure the overall behavior of the system, including model selection and experiment settings.

Parameter Value Description
model_name "gpt2" The foundation model being used for this experiment. GPT-2 is a transformer-based language model developed by OpenAI, capable of generating coherent text.
model_dir "" Directory where the model files are stored. An empty string indicates that the model will be instruction tuned as part of the experiment.
experiment_dir "./" Directory where experiment outputs, logs, will be stored. The current directory ("./" or "./") is used here.
problem "clustering" The optimization problem being solved. The configuration is set up for "clustering" by default, but also contains parameters for "mst" and "line_scheduling".
algorithm "all" Indicates which algorithms to run. The value "all" means that all implemented algorithms (best_of_llm, best_of_alg, top_ift) will be executed and compared.

3. Problem Parameters

These parameters define the specific characteristics of each optimization problem.

3.1 clustering

Clustering involves grouping a set of points so that points in the same group are more similar to each other than to those in other groups.

Parameter Value Description
num_points 24 Number of data points to be clustered in each problem instance.
r 10 Distances are either 1 or r.
num_test_points 50 Number of test instances to generate for evaluating the clustering performance.

3.2 mst

The Minimum Spanning Tree problem aims to find a subset of edges that forms a tree including every vertex, where the total weight of all the edges is minimized.

Parameter Value Description
num_lines 4 Number of horizontal/vertical lines in the grid graph
prob 0.4 Edge probability.
deg 2 The degree constraint parameter.
num_test_points 50 Number of test instances to generate for evaluating the performance.

3.3 line_scheduling

Line scheduling involves organizing optimizing visit times to minimize wait time.

Parameter Value Description
num_points 10 Number of points in each problem instance.
travel_time_range [1, 20] Range of possible travel times.
box_constraint_range [1, 20] Range of possible visit time constraints.
num_test_points 50 Number of test instances to generate for evaluating line scheduling performance.

5. Algorithm Parameters

These parameters control the behavior of different optimization algorithms implemented in the system.

5.1 best_of_llm

This algorithm generates multiple solutions using a language model and selects the best one according to the algorithm's objective.

Problem Parameter Value Description
clustering num_to_sample 1920 Number of candidate solutions to sample from the language model.
mst num_to_sample 600 Number of candidate solutions to sample for the mst problem.
line_scheduling num_to_sample 384 Number of candidate solutions to sample for line scheduling problem.

5.2 best_of_alg

Sample multiple samples from the feasible polytope and and select the best result according to the LLM's log probability.

Problem Parameter Value Description
clustering num_to_sample 1920 Number of algorithm runs or samples to generate for clustering.
mst num_to_sample 600 Number of algorithm runs or samples to generate for MST problem.
line_scheduling num_to_sample 384 Number of algorithm runs or samples to generate for line scheduling problem.
